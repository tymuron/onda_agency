# Agency Site Architect

## Why this exists

Web/design/dev services are maximally commoditized — every competitor says
"premium, fast, custom, results-driven", so the buyer's only remaining axis is
price. AI-assisted copy made this worse: most service sites now read identically
and machine-generated. This skill turns a generic site into one that a specific
buyer cannot confuse with anyone else, structured the way buyers actually scan
and decide, written in a human voice — which is simultaneously what converts and
what makes Google/AI cite you as *the* answer for a niche query.

Depth lives in five reference files (progressive disclosure). This file is the
operating procedure + the non-negotiable decisions.

## Operating procedure

1. **Positioning first — everything downstream depends on it.** Do not write a
   word of copy until you've run the positioning step in
   `references/positioning.md` (Dunford's 5 components + niche decision). For a
   solo/boutique operator the winning shape is almost always
   **`[Outcome] for [specific vertical] in [geography]`**, founder visible,
   proof-led, price published, with a point of view. Confirm/choose the niche
   with the user if ambiguous; otherwise infer from who they actually sell to
   and state the assumption.
2. **Lock the information architecture** from `references/structure-ia.md`: the
   canonical section order, multi-page vs one-page, nav, CTA strategy. The
   evidence is strong and specific — follow it, don't re-derive.
3. **Rewrite every line through the voice rules** in
   `references/copywriting-voice.md`. Run the AI-slop kill-list (below) over
   100% of copy. Message-mine real customer language where available.
4. **Rebuild the proof** per `references/portfolio-casestudies.md`: work
   surfaced high, results-led case studies, concept work *labelled honestly*,
   images as real indexable `<img>`.
5. **Plan the content** with `references/content-seo-geo-plan.md`: the
   service/vertical/location page map + blog, scoped for the operator's
   capacity, doorway-safe, build-ordered by fastest lead impact.
6. **Hand technical/GEO mechanics to the `seo-geo-audit` skill** (canonical,
   hreflang, schema, Core Web Vitals, crawl). This skill decides *what to say
   and where*; that one ensures it's *findable and fast*.
7. **Deliver / execute** in the output format below. Tailor to the operator's
   real capacity — a solo studio cannot run a 30-page content engine; give the
   version they can actually ship, in build order.

## Canonical homepage section order (follow unless there's a strong reason)

Evidence (NN/g, Orbit Media/Crestodina, CXL, Baymard, Unbounce): buyers judge
credibility in ~50ms, decide stay/leave in ~10s, scan not read, and self-qualify
~70-80% before contacting. They want, in order: *what you do + for whom*, *proof
of work*, *who you are*, *what it costs*. So:

1. Sticky header — logo left, ≤5 nav items, one distinct primary CTA top-right, language switcher top-right (not in nav)
2. **Hero** — outcome H1 (who-for + result + mechanism), one-line subhead, ONE primary CTA + one low-friction secondary (WhatsApp), an embedded proof token. No fake-urgency badges.
3. **Proof strip** — real client names/logos/result/quote, immediately, before doubt accumulates
4. **The work / portfolio — HIGH (top third)**, not buried. 4-6 strong, vertical-filterable case cards. For a design studio the work *is* the pitch.
5. Services — concrete, outcome-framed, each ends in a CTA
6. **About / who does this — mid-page**, with a real photo + first-person story (a one-line "built personally by X" also rides in the hero/proof). For a solo operator identity is a primary conversion asset (About is the #2-visited page).
7. Process — 3-4 plain steps (de-risks the buyer)
8. **Pricing / investment signal — present, not hidden** ("Projects from €X"). Missing price is a top agency mistake.
9. Testimonials — full, named, photo; also place one *adjacent to the final CTA* (peak-doubt moment)
10. FAQ — objection-killer (timeline, why solo, what's included, languages)
11. Final CTA band — restate value + primary CTA + WhatsApp + a testimonial beside it
12. Footer as navigation — full sitemap, contact, languages

Rules: proof appears **≥4×**; **every section ends with a path forward** (no
CTA-less sections); one primary action repeated (multiple CTAs to the *same*
destination are fine; competing destinations cut conversion); clicks-to-contact
≤1 from any scroll position (sticky CTA + WhatsApp). Use a **hybrid**: this
sequenced homepage *plus* dedicated indexed pages (`/work`, `/services` or
per-vertical, `/about`, `/contact`, `/blog`) for SEO depth.

## AI-slop kill-list (run over 100% of copy — this is usually the user's pain)

Delete/rewrite on sight:
- **Words:** elevate, unlock, empower, seamless, robust, leverage, utilize,
  holistic, cutting-edge, best-in-class, synergy, streamline, reimagine,
  crafted, bespoke (when empty), "solutions", "digital landscape/age",
  "in today's…", "we are passionate about", "your trusted partner",
  "take it to the next level", "drive results".
- **Structures:** rule-of-three triads ("fast, beautiful, reliable"); "not
  just X, but Y" / "it's not…it's…"; "-ing" tail clauses ("…helping
  businesses grow"); hollow frames ("built to…", "designed to deliver");
  hedging ("may help", "can potentially"); em-dash overuse (≤1 per ~150 words).
- **Moves:** all-"we"/no-"you"; **fake urgency/scarcity** ("Taking 3 new
  projects for April", "Only 2 spots left", "Quote in 24 hours" badges);
  vague adjective with no number; any claim with no proof; any sentence true
  of any company in any industry.
- **Solo operators write "I", never "we".** Owning "it's just me — you work
  directly with the person who builds it" is a *selling point*, not a weakness.
- **Three tests on every line:** (1) Read-aloud — would you say this to a
  client over coffee? (2) "So what?" — chase to a concrete buyer outcome.
  (3) "Could a competitor paste this unchanged?" — if yes, add specifics
  until no.

## Hero formula

`[Specific outcome] for [specific buyer] — [mechanism/differentiator].`
Subhead: who it's for + the founder mechanism. Plus a real proof line. No
cleverness in the H1 (NN/g: users reject hype; objective+concise+scannable copy
tested +124% usability).
- ❌ "Premium websites for businesses that need to look serious."
- ✅ "Websites that turn visitors into booked patients — for dental & aesthetic
  clinics in Spain." / sub: "I'm Tymur. I design and build the whole site
  myself — no agency layers. Fixed price, live in ~3 weeks."

## Output format

```
# Site Architecture & Rebuild Plan — <site>

## Positioning
Chosen niche + Dunford one-liner + the point of view. The hero H1/subhead (final copy).

## Information architecture
The page list + the homepage section order (final), nav, CTA strategy, trilingual handling.

## Copy
Section-by-section final copy (or the rewrite of each existing section), in the
human voice, with EN and localized variants where relevant. AI-slop removed.

## Proof
Case-study template + how concept work is labelled + image/indexability plan.

## Content & SEO/GEO plan
The service/vertical/location page map + blog post list (titled, EN+ES) +
build order + doorway-safety notes. (Defer technical/CWV to seo-geo-audit.)

## Build order
Numbered, fastest-lead-impact first, scoped to the operator's real capacity.
```

## Guardrails

- Never fabricate proof: no invented clients, fake logos, fake testimonials,
  fake scarcity. Concept/spec work must be labelled "Concept" at the top of the
  piece (fabricated testimonials are FTC fraud; one exposed fake destroys all
  credibility). Build credibility ethically (concept → real teardown → free/
  discounted real build → named result).
- Don't water positioning back to "we help all businesses" — that's the failure
  mode you're fixing. Hold the niche in the hero; serve breadth on separate
  pages.
- Match the plan to real capacity and stack. Bias to fewer, deeper, genuinely
  useful pages over volume (Google people-first; doorway policy).
- Explain the *why* (cite the mechanism/source) so the operator can make their
  own judgement calls later, not just follow rules.
