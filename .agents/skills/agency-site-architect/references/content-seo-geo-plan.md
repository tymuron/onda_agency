# SEO/GEO Content Plan for a Service/Agency Site

What pages/posts to create so Google ranks it and AI engines recommend it.
(Technical/CWV/hreflang mechanics → seo-geo-audit skill.)

## 1. Content architecture (5 page classes, hub-and-spoke)
- **Service pages (BOFU)** — one per *core service* (Web Design, Custom/Webflow
  Dev, Website Redesign, SEO/Performance, Branding). Conversion hubs.
- **Vertical/industry pages (BOFU, highest ROI)** — "web design for dental
  clinics", "…for restaurants", "…for real estate", "…for aesthetic clinics",
  "…for architecture studios". Out-convert generic service pages (exact buyer
  language + SERP intent).
- **Location pages (BOFU, geo)** — small hand-written set, only markets truly
  served (Barcelona, Madrid, Valencia, Marbella/Costa del Sol, + an EN
  international/remote page).
- **Comparison/alternative pages (BOFU, decision)** — agency vs freelancer;
  Webflow vs WordPress; custom vs Wix/Squarespace. Highest-converting blog
  format (3 BOFU posts can beat 50 TOFU).
- **Blog/resource hub (TOFU→MOFU)** — guides, data, tools, checklists that earn
  links + AI citations, funneling into money pages.
Interlink as topical clusters: each service/vertical = hub; spokes (posts,
FAQs, case studies) link up with descriptive anchors; hub links down + across
to case studies. Path on every page: blog → vertical/service → matching case
study → contact/quote. Internal links = micro-CTAs placed where motivation
peaks.

## 2. Service × vertical × location matrix — scoped for a solo operator
Do NOT build the full Cartesian matrix (doorway farm). Year-one cap ≈ 25-35
indexable money/vertical/location pages: ~5 service pages, ~5 national vertical
pages (carry most commercial weight), ~4-6 hand-written location pages, and
vertical×top-city pages ONLY after the national vertical page ranks and real
volume + real local proof exist. "Ten strong pages beat 100 weak ones."

## 3. Doorway-avoidance (HARD constraints — each vertical/location page)
Must satisfy ALL: (1) ≥60-70% unique body vs siblings (not name-swaps);
(2) real specific proof (named case study/screenshot/testimonial for that
segment/city); (3) segment-specific substance (dental: online booking,
treatment pages, patient trust; restaurants: menu/reservations/Maps; real
estate: listings/lead capture); (4) vertical/local-specific FAQ (3-6 real
questions); (5) distinct intro/H2s/CTA/meta; (6) genuine service of that area
(no fake addresses); (7) in browseable nav, not orphaned; (8) schema
LocalBusiness/Service/FAQPage/Review. If a page can't meet 1-6 with real
material, **don't publish it.**

## 4. Blog topics that rank AND convert (titles, EN + ES)
**BOFU money posts (build FIRST — fastest leads):**
- "How much does a website cost in Spain? (2026 pricing guide)" /
  "¿Cuánto cuesta una página web en España? (Guía de precios 2026)" — highest-
  intent term in the niche.
- "How to choose a web designer: 12 questions to ask" / "Cómo elegir un
  diseñador web: 12 preguntas antes de contratar".
- "Agency vs freelance web designer" / "Agencia vs freelance de diseño web".
- "Webflow vs WordPress for small businesses (2026)" / "Webflow vs WordPress
  para pymes".
- "Wix/Squarespace vs a custom website: 3-year cost" / "Wix vs web a medida:
  coste real a 3 años".
- "Signs your website needs a redesign" / "Señales de que tu web necesita un
  rediseño".
**Vertical commercial posts (one per vertical → links to vertical page + case
study):** dental ("…what a clinic site needs to win patients"), restaurants
("menu, reservations & Google Maps"), real estate ("…that generates qualified
leads"), aesthetic clinics ("trust, booking, before/after"), architecture
("portfolio that wins projects") — EN + ES.
**TOFU link/citation earners:** "Small business website checklist (2026)" +
PDF; a free "website cost calculator"; original data **"State of SMB websites
in Spain 2026"** (survey/audit — links + AI-citation magnet); "Core Web Vitals
in plain English".

## 5. Keyword strategy for this niche
Seeds: `diseño web [vertical]`, `página web para [vertical]`, `precio/coste
página web`, `cuánto cuesta una página web`, `diseño web [ciudad]`, `agencia
diseño web [ciudad]`, `[builder] vs [builder]`, `rediseño web`. One primary
intent per URL (transactional → service/vertical/location/comparison;
commercial-investigation "best/vs/cuánto cuesta" → BOFU blog; informational
"qué es/cómo" → TOFU). New low-authority site: target Ahrefs KD <20 / Semrush
<30 (ideally 0-14) and validate the live SERP manually; long-tail+local+vertical
can move in 8-12 weeks; head terms ("diseño web") are multi-year — don't chase
early. ES = money language for Spain; EN serves international/expat + AI English
queries.

## 6. Formats that earn AI citations & links
Answer-first (H2 question → 40-60-word direct answer → detail); predictable
modules (Definition, Steps/Checklist, Pros/Cons, Comparison **table**, price
ranges with figures, FAQ, Sources). FAQPage schema ≈ higher citation rate;
stats+sources+tables boost AI visibility materially; AI traffic converts well.
Build 3 linkable assets: cost calculator, downloadable checklist/template, one
original data study.

## 7. Programmatic vs doorway — the line
Programmatic is fine only with genuinely unique data/expertise/proof per URL +
browseable hierarchy. No unique dataset → manual, small, deep set; a solo
operator's "programmatic" = shared template structure with hand-written unique
content per vertical/city.

## 8. Internal linking & topical authority
Pillars: a "Web Design" pillar + an "SEO/Performance" pillar (2-4k words);
clusters hang off each. spoke→hub (exact-topic anchor), hub→all spokes,
vertical post→vertical page→that vertical's case study→contact, comparison
post→service page. Money pages ≤2 clicks from home.

## 9. Cadence & build order (solo, fastest lead impact)
1. **Wk 1-3:** 5 service pages + pricing post (ES+EN) + "how to choose" +
   "agency vs freelance" (BOFU first = fastest leads).
2. **Wk 4-8:** 5 vertical pages (ES primary) + 1 vertical commercial post each
   + ensure 1 case study per vertical.
3. **Wk 9-12:** comparison posts + cost calculator + checklist asset.
4. **Mo 4-6:** 4-6 location pages, then vertical×city where data+proof exist;
   launch the original data study.
Steady: ~2-4 posts/month, 70% new / 30% refresh (pricing/data monthly,
"best"/comparison quarterly, guides 6-monthly, evergreen annually — never bump
dates without substantive edits). Measure content→lead via per-page
contact/quote conversions, assisted conversions, AI-referral sessions.

## 10. Trilingual
ES = full money set (all service/vertical/location/BOFU). EN = parallel set for
international/expat + AI English citations (localize, don't translate). KA =
brand/about + 1-2 flagship pages only. Bidirectional self-referential hreflang;
only index localized versions; never thin auto-translations (scaled-content
risk).
